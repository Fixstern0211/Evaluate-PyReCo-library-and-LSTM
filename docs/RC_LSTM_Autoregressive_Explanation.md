# Reservoir Computing vs LSTM: Autoregressive Multi-step Prediction

This document explains how RC and LSTM models perform multi-step prediction using the autoregressive approach.

---

## 1. Multi-step Prediction Strategy

Professor recommended **Approach 1: Autoregressive (Free-run) Iteration**.

### How it works

- Train the model with `n_out = 1` (single-step prediction)
- At inference, iteratively feed predictions back as input:

```
Step 1: [x(0), x(1), ..., x(99)]   → predict x̂(100)
Step 2: [x(1), x(2), ..., x̂(100)] → predict x̂(101)  (using predicted x̂(100))
Step 3: [x(2), x(3), ..., x̂(101)] → predict x̂(102)
...
Step k: [x(k-1), ..., x̂(k+98)]    → predict x̂(k+99)
```

### Why this approach

- Only one model needed for any prediction horizon
- Both RC and LSTM can use the same approach → **fair comparison**
- Generic and flexible

## 2. RC Output Computation

**The output is computed from the last reservoir state.**

### Reservoir State Evolution

For an input window of `n_in = 100` timesteps:

```
Input sequence: [x(0), x(1), x(2), ..., x(99)]
                   ↓      ↓      ↓          ↓
Reservoir states: s(0) → s(1) → s(2) → ... → s(99)
                                              ↓
                                        Readout layer (W_out)
                                              ↓
                                      Output: y_pred(100)
```

### State Update Formula

Each state s(t) depends on the current input and previous state:

```
r(t+Δt) = (1-α)·r(t) + α·tanh(A·r(t) + W_in·u(t) + ξ·1)
```

Where:
- **α**: leakage rate, controls how fast node weights update
- **r**: reservoir state vector, records activation of each node
- **A**: weighted adjacency matrix (sparse, generated by Erdős-Rényi)
- **W_in**: N×M matrix, maps M-dimensional input to reservoir
- **u(t)**: input signal (M-dimensional)
- **ξ**: bias

### Why Last State Contains All Information

The final state `s(99)` implicitly encodes the entire input history through recursive dynamics:

```
s(99) = f(s(98), x(99))
      = f(f(s(97), x(98)), x(99))
      = f(f(f(...f(s(0), x(1))...), x(98)), x(99))
      = F(x(0), x(1), x(2), ..., x(99))   ← Function of ALL inputs!
```

**Analogy: Snowball Effect**
```
Input sequence: x(0), x(1), x(2), ..., x(99)
                  ↘    ↘    ↘           ↘
State evolution:  ⚪ → ⚪ → ⚪ → ... → 🔵 (s(99))
                 small  medium  large    Contains all information
```

## 3. Weight Matrices in RC

### Overview

| Matrix | Dimensions | Role | Trained? |
|--------|------------|------|----------|
| **W_in** | N × M | Input → Reservoir | Random, **fixed** |
| **A** | N × N | Reservoir internal connections | Random, **fixed** |
| **W_out** | P × N | Reservoir → Output | **Trained** (only this!) |

### Dimension Definitions

- **M**: Input dimension (determined by data)
  - Lorenz: M = 3 (x, y, z)
  - Mackey-Glass: M = 1
  - Santa Fe: M = 1

- **N**: Number of reservoir nodes (hyperparameter, you set this)
  - Typical values: 50, 100, 200, 500

- **P**: Output dimension (determined by prediction target)
  - Same as M for time series prediction

### Data Flow

```
Input(M) ──W_in──→ Reservoir(N nodes) ──W_out──→ Output(P)
                        ↺ A (internal connections)
```

## 4. Computing Output from Last State

### The State Vector s(t)

Reservoir has N nodes, each with an activation value:

```
N = 100 nodes

s(t) = ┌       ┐
       │  0.23 │  ← node 1 activation
       │ -0.87 │  ← node 2 activation
       │  0.54 │  ← node 3 activation
       │  ...  │
       │ -0.12 │  ← node 100 activation
       └       ┘

       (N × 1) = (100 × 1)
```

### Output Calculation: Matrix Multiplication

```
Output = W_out × s(t)

(P×1) = (P×N) × (N×1)
```

### Concrete Example (Lorenz: P=3, N=100)

```
W_out (3×100)           s(99) (100×1)           y_pred (3×1)

┌                      ┐   ┌      ┐              ┌        ┐
│ w₁,₁  w₁,₂ ... w₁,₁₀₀ │   │ s₁   │              │ x(100) │
│ w₂,₁  w₂,₂ ... w₂,₁₀₀ │ × │ s₂   │      =       │ y(100) │
│ w₃,₁  w₃,₂ ... w₃,₁₀₀ │   │ ...  │              │ z(100) │
└                      ┘   │ s₁₀₀ │              └        ┘
                           └      ┘

     (3×100)             (100×1)                  (3×1)
```

Each output component:
```
x(100) = w₁,₁·s₁ + w₁,₂·s₂ + ... + w₁,₁₀₀·s₁₀₀  (row 1 × state vector)
y(100) = w₂,₁·s₁ + w₂,₂·s₂ + ... + w₂,₁₀₀·s₁₀₀  (row 2 × state vector)
z(100) = w₃,₁·s₁ + w₃,₂·s₂ + ... + w₃,₁₀₀·s₁₀₀  (row 3 × state vector)
```

## 5. RC vs LSTM Comparison

### Core Difference: What Gets Trained

| | RC | LSTM |
|---|---|---|
| Internal weights | Random, **fixed** | **Trained** (backpropagation) |
| Output weights | **Trained** (Ridge regression) | **Trained** |
| Training method | Solve linear equation (fast) | Gradient descent (slow) |

### Output Mechanism: Same Principle

Both use a linear layer to map internal state to output:

| | RC | LSTM |
|---|---|---|
| State vector | s(t) - reservoir state | h(t) - hidden state |
| State dimension | N (num_nodes) | H (hidden_size) |
| Output calculation | y = W_out × s(t) | y = W_out × h(t) |

```
RC:
Input → [Fixed W_in] → [Fixed Reservoir] → s(t) → [Trained W_out] → Output
                                           (N×1)    (P×N)           (P×1)

LSTM:
Input → [Trained Gate Networks] → h(t) → [Trained W_out] → Output
         (complex, all trained)   (H×1)   (P×H)            (P×1)
```

### Autoregressive Prediction: Identical for Both

```python
def autoregressive_predict(model, initial_input, n_steps):
    """Works for both RC and LSTM"""
    current_input = initial_input.copy()
    predictions = []

    for step in range(n_steps):
        pred = model.predict(current_input)   # Single-step prediction
        predictions.append(pred)

        # Sliding window: remove oldest, append prediction
        current_input = concat(current_input[1:], pred)

    return predictions
```

## 6. Why RC Trains Faster

### Traditional Neural Network (LSTM)
- Train **all** weights (W_in, W_hidden, W_out)
- Requires backpropagation, gradient computation
- Iterative optimization over many epochs

```python
# LSTM training: multiple iterations
for epoch in range(100):
    loss = forward(x)
    loss.backward()        # Backpropagation
    optimizer.step()       # Update all weights
```

### Reservoir Computing
- Only train **W_out** (one linear layer)
- Use Ridge regression (solve linear equation directly)
- **No backpropagation needed**

```python
# RC training: one-shot solution
states = reservoir.run(x)           # Fixed weights, forward only
W_out = ridge_regression(states, y) # Direct solution, no iteration
```

## 7. Model Persistence

### Saving and Loading

```python
import pickle

# Save trained model
with open('rc_model.pkl', 'wb') as f:
    pickle.dump(model, f)

# Load model (can be on different machine)
with open('rc_model.pkl', 'rb') as f:
    model = pickle.load(f)

# Use directly, no retraining needed
predictions = model.predict(X_new)
```

### What Gets Saved

| Content | Saved |
|---------|-------|
| W_in (input weights) | ✓ |
| A (reservoir connections) | ✓ |
| W_out (trained output weights) | ✓ |
| Hyperparameters | ✓ |

## 8. Summary

| Aspect | RC | LSTM |
|--------|-----|------|
| Training | Only W_out (fast) | All weights (slow) |
| State | s(t), N×1 | h(t), H×1 |
| Output | W_out × s(t) | W_out × h(t) |
| Autoregressive | ✓ Same method | ✓ Same method |
| Multi-step prediction | Feed predictions back | Feed predictions back |
| Fair comparison | ✓ | ✓ |

**Key Insight**: Both models can use identical autoregressive prediction, making the comparison fair. The only difference is the internal architecture and training method.
